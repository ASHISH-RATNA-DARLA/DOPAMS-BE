KEY FINDINGS
================================================================================

SCHEMA CHANGES:
  - 2 tables removed (brief_facts_drugs, dedup_comparison_progress_backup)
  - 6 tables added (drug_categories, drug_ignore_list, geo_*, ir_pending_fk, etc.)
  - 4 tables modified with column changes

COLUMN CHANGES:
  - accused.accused_status (NEW) - holds status information
  - brief_facts_drug (RESTRUCTURED) - 10 columns removed, 10 added
    * Old: standardized_weight_kg, quantity_numeric, etc.
    * New: weight_kg, weight_g, volume_ml, volume_l, count_total, raw_*, etc.
  - crimes.additional_json_data (NEW) - for extensibility

CRITICAL CHANGES:
  1. accusedStatus field:
     - Now uses COALESCE(bfa.status, a.accused_status, 'Unknown')
     - Priority: brief_facts_accused.status > accused.accused_status > 'Unknown'
  
  2. drugWithQuantity field:
     - Restructured from old columns to new ones
     - JSONB object built from: primary_drug_name, weight_kg, volume_ml, count_total, seizure_worth
     - Frontend receives same JSONB structure

COMPATIBILITY LAYER:
  - Materialized Views (MVs) act as transformation layer
  - 4 views created: accuseds_mv, firs_mv, advanced_search_accuseds_mv, advanced_search_firs_mv
  - Views handle all schema-to-API transformations
  - Frontend code requires NO CHANGES

CRITICAL SUCCESS FACTORS
================================================================================

1. MATERIALIZED VIEWS MUST BE CREATED
   - File: MVs_newdb.sql
   - Contains 4 views with proper JOIN and transformation logic
   - Creates indexes for performance

2. DATA MUST BE MIGRATED CORRECTLY
   - accused.accused_status must be populated
   - brief_facts_drug new columns must contain data
   - Foreign key relationships maintained

3. VIEWS MUST BE INDEXED
   - Index on id, crimeId, year, unit, ps, etc.
   - Performance depends on indexes
   - Indexes included in MVs_newdb.sql

4. STATUS RESOLUTION MUST WORK
   - COALESCE chain must be correct
   - Order of priority: bfa.status > a.accused_status > 'Unknown'
   - Fallback to 'Unknown' for null values

5. DRUG MAPPING MUST BE ACCURATE
   - New columns: weight_kg, weight_g, volume_ml, volume_l, count_total
   - JSONB object construction in view
   - Seizure worth included in output
